Chapter 3 - Generative Models for Discrete Data

For generative classifiers, to classify a feature vector x\mathbf{x}x, we first compute the class-conditional density and then apply the Bayes rule:

p(y=c∣x,θ)∝p(x∣y=c,θ)p(y=c∣θ)p(y=c | \mathbf{x}, \boldsymbol{\theta}) \propto p(\mathbf{x} | y=c, \boldsymbol{\theta}) p(y=c | \boldsymbol{\theta})p(y=c∣x,θ)∝p(x∣y=c,θ)p(y=c∣θ)

For this chapter, we assume x\mathbf{x}x is made of discrete symbols.

Example 1 : Beta - Bernoulli Distribution

Suppose Xi∼Ber⁡(θ)X_{i} \sim \operatorname{Ber}(\theta)Xi​∼Ber(θ) with Xi=1X_i=1Xi​=1 represents head (success). θ\thetaθ is the rate parameter that captures the probability of success. We are required to estimate θ\thetaθ.

Likelihood:
If the data (of trials) are iid, the likelihood has the form:
p(D∣θ)=θN1(1−θ)N0p(\mathcal{D} | \theta)=\theta^{N_{1}}(1-\theta)^{N_{0}}p(D∣θ)=θN1​(1−θ)N0​
where, N1N_1N1​ is the number of heads in the data and N0N_0N0​ is the number of tails. Total number of trials, N=N1+N0N = N_1 + N_0N=N1​+N0​

We can model N1N_1N1​ as the number of success in NNN trials using a binomial distribution: N1∼Bin⁡(N,θ)N_{1} \sim \operatorname{Bin}(N, \theta)N1​∼Bin(N,θ) whose pmf is of the form:
Bin⁡(N1∣N,θ)≜(NN1)θN1(1−θ)N−N1\operatorname{Bin}(N_1 | N, \theta) \triangleq \left( \begin{array}{l}{N} \\ {N_1}\end{array}\right) \theta^{N_1}(1-\theta)^{N-N_1}Bin(N1​∣N,θ)≜(NN1​​)θN1​(1−θ)N−N1​

Prior:
θ\thetaθ is a probability and we need its prior to have support over the interval [0, 1].
To make the math easier, it would convenient if the prior had the same form as the likelihood:
p(θ)∝θγ1(1−θ)γ2p(\theta) \propto \theta^{\gamma_{1}}(1-\theta)^{\gamma_{2}}p(θ)∝θγ1​(1−θ)γ2​
for some parameters γ1\gamma_1γ1​ and γ2\gamma_2γ2​. For this case, we can easily compute the posterior (likelihood x prior):
p(θ∣D)∝p(D∣θ)p(θ)=θN1(1−θ)N0θγ1(1−θ)γ2=θN1+γ1(1−θ)N0+γ2p(\theta | \mathcal{D}) \propto p(\mathcal{D} | \theta) p(\theta)=\theta^{N_{1}}(1-\theta)^{N_{0}} \theta^{\gamma_{1}}(1-\theta)^{\gamma_{2}}=\theta^{N_{1}+\gamma_{1}}(1-\theta)^{N_{0}+\gamma_{2}}p(θ∣D)∝p(D∣θ)p(θ)=θN1​(1−θ)N0​θγ1​(1−θ)γ2​=θN1​+γ1​(1−θ)N0​+γ2​

  • When the prior and the posterior have the same form, we say that the prior is a conjugate prior for the corresponding likelihood. Conjugate priors are widely used because they simplify computation, and are easy to interpret.

In the case of the Bernoulli, the conjugate prior is the beta distribution of the form:
Beta⁡(θ∣a,b)∝θa−1(1−θ)b−1\operatorname{Beta}(\theta | a, b) \propto \theta^{a-1}(1-\theta)^{b-1}Beta(θ∣a,b)∝θa−1(1−θ)b−1

If we know “nothing” about θθθ, except that it lies in the interval [0, 1], we can use a uniform prior, which is a kind of uninformative prior. This can be obtained by setting a=b=1a=b=1a=b=1 for the beta distribution.

Posterior:
Multiply binomial likelihood and beta prior to get beta posterior:
p(θ∣D)∝Bin⁡(N1∣θ,N0+N1)∗Beta⁡(θ∣a,b)∝Beta⁡(θ∣N1+a,N0+b)p(\theta | \mathcal{D}) \quad \propto \quad \operatorname{Bin}\left(N_{1} | \theta, N_{0}+N_{1}\right) * \operatorname{Beta}(\theta | a, b) \propto\operatorname{Beta}\left(\theta | N_{1}+a, N_{0}+b\right)p(θ∣D)∝Bin(N1​∣θ,N0​+N1​)∗Beta(θ∣a,b)∝Beta(θ∣N1​+a,N0​+b)

Now, that we know the posterior distribution, we can determine the posterior mean and posterior variance.

Note that the posterior can be updated sequentially (k updates) as new datapoints arrive, or in a batch fashion (1 update of size k) - Both are equivalent and this makes Bayesian inference particularly well-suited to online learning.

Posterior predictive distribution:
Consider the task of making a prediction of future observable data. What is the probability of heads in a single future trial under a Beta(a, b) posterior?

p(x~=1∣D)=∫01p(x=1∣θ)p(θ∣D)dθ=∫01θBeta⁡(θ∣a,b)dθ=E[θ]=aa+b\begin{aligned} p(\tilde{x}=1 | \mathcal{D}) &amp;=\int_{0}^{1} p(x=1 | \theta) p(\theta | \mathcal{D}) d \theta \\ &amp;=\int_{0}^{1} \theta \operatorname{Beta}(\theta | a, b) d \theta=\mathbb{E}[\theta]=\frac{a}{a+b} \end{aligned}p(x~=1∣D)​=∫01​p(x=1∣θ)p(θ∣D)dθ=∫01​θBeta(θ∣a,b)dθ=E[θ]=a+ba​​

Note: p(x=1∣θ)=θp(x=1 | \theta) = \thetap(x=1∣θ)=θ since by definition θ\thetaθ is the probability of success.

Example 2 : Dirichlet - Multinomial Distribution

In the previous case, we only has two possible outcomes. Here, we consider K possible outcomes (rolling a dice), that will be modelled by a Multnomial distribution.

Likelihood:
Suppose we observe NNN dice rolls, D={x1,…,xN}\mathcal{D}=\left\{x_{1}, \dots, x_{N}\right\}D={x1​,…,xN​}, where xi∈{1,…,K}x_{i} \in\{1, \ldots, K\}xi​∈{1,…,K}. If we assume the data is iid, the likelihood has the form:
p(D∣θ)=∏k=1KθkNkp(\mathcal{D} | \boldsymbol{\theta})=\prod_{k=1}^{K} \theta_{k}^{N_{k}}p(D∣θ)=k=1∏K​θkNk​​
where, NkN_{k}Nk​ is the number of times event k occured. Note that θ\bm \thetaθ is a vector of probabilities, where the kkkth position corresponds to the prob of success of kkkth side of the dice.

Prior:
Since the parameter vector lives in the K-dimensional probability simplex, we need a prior that has support over this simplex. Ideally it would also be conjugate(prior has same form as the posterior). Fortunately, the Dirichlet distribution satisfies both criteria.
Dir⁡(θ∣α)=1B(α)∏k=1Kθkαk−1\operatorname{Dir}(\boldsymbol{\theta} | \boldsymbol{\alpha})=\frac{1}{B(\boldsymbol{\alpha})} \prod_{k=1}^{K} \theta_{k}^{\alpha_{k}-1} Dir(θ∣α)=B(α)1​k=1∏K​θkαk​−1​

Posterior:
Multiplying the likelihood by the prior, we find that the posterior is also Dirichlet.
p(θ∣D)∝p(D∣θ)p(θ)∝∏k=1KθkNkθkαk−1=∏k=1Kθkαk+Nk−1=Dir⁡(θ∣α1+N1,…,αK+NK)\begin{aligned} p(\boldsymbol{\theta} | \mathcal{D}) &amp; \propto p(\mathcal{D} | \boldsymbol{\theta}) p(\boldsymbol{\theta}) \\ &amp; \propto \prod_{k=1}^{K} \theta_{k}^{N_{k}} \theta_{k}^{\alpha_{k}-1}=\prod_{k=1}^{K} \theta_{k}^{\alpha_{k}+N_{k}-1} \\ &amp;=\operatorname{Dir}\left(\boldsymbol{\theta} | \alpha_{1}+N_{1}, \ldots, \alpha_{K}+N_{K}\right) \end{aligned}p(θ∣D)​∝p(D∣θ)p(θ)∝k=1∏K​θkNk​​θkαk​−1​=k=1∏K​θkαk​+Nk​−1​=Dir(θ∣α1​+N1​,…,αK​+NK​)​

Posterior predictive distribution:
For a a single multinoulli trial, what is the probability that the dice has side jjj up (out of K possible sides).

p(X=j∣D)=∫p(X=j∣θ)p(θ∣D)dθ=∫p(X=j∣θj)[∫p(θ−j,θj∣D)dθ−j]dθj=∫θjp(θj∣D)dθj=E[θj∣D]=αj+Nj∑k(αk+Nk)=αj+Njα0+N\begin{aligned} p(X=j | \mathcal{D}) &amp;=\int p(X=j | \boldsymbol{\theta}) p(\boldsymbol{\theta} | \mathcal{D}) d \boldsymbol{\theta} \\ &amp;=\int p\left(X=j | \theta_{j}\right)\left[\int p\left(\boldsymbol{\theta}_{-j}, \theta_{j} | \mathcal{D}\right) d \boldsymbol{\theta}_{-j}\right] d \theta_{j} \\ &amp;=\int \theta_{j} p\left(\theta_{j} | \mathcal{D}\right) d \theta_{j}=\mathbb{E}\left[\theta_{j} | \mathcal{D}\right]=\frac{\alpha_{j}+N_{j}}{\sum_{k}\left(\alpha_{k}+N_{k}\right)}=\frac{\alpha_{j}+N_{j}}{\alpha_{0}+N} \end{aligned}p(X=j∣D)​=∫p(X=j∣θ)p(θ∣D)dθ=∫p(X=j∣θj​)[∫p(θ−j​,θj​∣D)dθ−j​]dθj​=∫θj​p(θj​∣D)dθj​=E[θj​∣D]=∑k​(αk​+Nk​)αj​+Nj​​=α0​+Nαj​+Nj​​​

MLE vs MAP

MLE vs MAP: the connection between Maximum Likelihood and Maximum A Posteriori Estimation - Agustinus Kristiadi’s Blog

  • Maximum Likelihood Estimation (MLE) and Maximum A Posteriori (MAP), are both a method for estimating some variable in the setting of probability distributions or graphical models. They are similar, as they compute a single estimate, instead of a full distribution.

  • MLE: take for example, when fitting a Gaussian to our dataset, we immediately take the sample mean and sample variance, and use it as the parameter of our Gaussian - these parameter estimates are based on MLE - the parameter values that maximize the likelihood of the data (can be obtained by maximizing log-likelihood):
    θMLE=arg⁡max⁡θP(X∣θ)=arg⁡max⁡θ∏iP(xi∣θ)\begin{aligned} \theta_{M L E} &amp;=\underset{\theta}{\arg \max } P(X | \theta) \\ &amp;=\underset{\theta}{\arg \max } \prod_{i} P\left(x_{i} | \theta\right) \end{aligned}θMLE​​=θargmax​P(X∣θ)=θargmax​i∏​P(xi​∣θ)​
    Since prob. values are [0,1], the product above can result in computation underflow. Better to work in the log space.
    θMLE=arg⁡max⁡θlog⁡P(X∣θ)=arg⁡max⁡θlog⁡∏iP(xi∣θ)=arg⁡max⁡θ∑ilog⁡P(xi∣θ)\begin{aligned} \theta_{M L E} &amp;=\underset{\theta}{\arg \max } \log P(X | \theta) \\ &amp;=\underset{\theta}{\arg \max } \log \prod_{i} P\left(x_{i} | \theta\right) \\ &amp;=\underset{\theta}{\arg \max } \sum_{i} \log P\left(x_{i} | \theta\right) \end{aligned}θMLE​​=θargmax​logP(X∣θ)=θargmax​logi∏​P(xi​∣θ)=θargmax​i∑​logP(xi​∣θ)​

  • MAP usually comes up in Bayesian setting. As the name suggests, it works on a posterior distribution, not only the likelihood, which can be obtained as product of likelihood and prior.
    θMAP=arg⁡max⁡θP(X∣θ)P(θ)=arg⁡max⁡θlog⁡P(X∣θ)P(θ)=arg⁡max⁡θlog⁡∏iP(xi∣θ)P(θ)=arg⁡max⁡θ∑ilog⁡P(xi∣θ)P(θ)\begin{aligned} \theta_{M A P} &amp;=\underset{\theta}{\arg \max } P(X | \theta) P(\theta) \\ &amp;=\underset{\theta}{\arg \max } \log P(X | \theta) P(\theta) \\ &amp;=\underset{\theta}{\arg \max } \log \prod_{i} P\left(x_{i} | \theta\right) P(\theta) \\ &amp;=\underset{\theta}{\arg \max } \sum_{i} \log P\left(x_{i} | \theta\right) P(\theta) \end{aligned}θMAP​​=θargmax​P(X∣θ)P(θ)=θargmax​logP(X∣θ)P(θ)=θargmax​logi∏​P(xi​∣θ)P(θ)=θargmax​i∑​logP(xi​∣θ)P(θ)​

  • Comparing both MLE and MAP equation, the only thing differs is the inclusion of prior P(θ)P(\theta)P(θ) in MAP, otherwise they are identical. What it means is that, the likelihood is now weighted with some weight coming from the prior.

  • If we assume a uniform prior, the terms in the summation above are equally weighted and its value can be taken out of the maximization as it acts as a constant.
    θMAP=arg⁡max⁡θ∑ilog⁡P(xi∣θ)P(θ)=arg⁡max⁡θ∑ilog⁡P(xi∣θ) const =arg⁡max⁡θ∑ilog⁡P(xi∣θ)=θMLE\begin{aligned} \theta_{M A P} &amp;=\underset{\theta}{\arg \max } \sum_{i} \log P\left(x_{i} | \theta\right) P(\theta) \\ &amp;=\underset{\theta}{\arg \max } \sum_{i} \log P\left(x_{i} | \theta\right) \text { const } \\ &amp;=\underset{\theta}{\arg \max } \sum_{i} \log P\left(x_{i} | \theta\right) \\ &amp;=\theta_{M L E} \end{aligned}θMAP​​=θargmax​i∑​logP(xi​∣θ)P(θ)=θargmax​i∑​logP(xi​∣θ) const =θargmax​i∑​logP(xi​∣θ)=θMLE​​
    That is to say, MLE is a special case of MAP, where the prior is uniform.

  • If we use different prior, say, a Gaussian, then our prior is not constant anymore, as depending on the region of the distribution, the probability is high or low, never always the same.

  • One can proof that in the limit of infinite data, both estimates converge.

  • Since in MLE, the parameter estimates are based on the current data alone, it is prone to overfitting. It does not tell us anything about the quality of the estimate, nor about how well we can actually predict anything (in future) from the estimates.

Naive Bayes classifiers (Notes Link)

  • Assume classification setting with discrete labels y∈{0,…,Ly−1}y \in\left\{0, \ldots, L_{y}-1\right\}y∈{0,…,Ly​−1}. Let Ly=2L_{y} = 2Ly​=2, i.e., binary classification.

  • x={x1,…,xK}\mathbf{x} = \left\{x_{1}, \dots, x_{K}\right\}x={x1​,…,xK​}, where each xkx_kxk​ can take on one of LkL_kLk​ labels xk∈{0,…,Lk−1}x_{k} \in \left\{0, \dots, L_{k}-1\right\}xk​∈{0,…,Lk​−1}.   • To build a probabilistic classification algorithm that actually optimizes a likelihood, we could use p(x,y∣θ)p(\mathbf{x}, y | \theta)p(x,y∣θ), which can be decomposed as a product of class conditional and prior.
    p(x,y∣θ)=p(x∣y,θ)p(y∣θ)p(\mathbf{x}, y | \theta)=p(\mathbf{x} | y, \theta) p(y | \theta)p(x,y∣θ)=p(x∣y,θ)p(y∣θ)

  • p(y∣θ)p(y | \theta)p(y∣θ) is a prior on yyy (we could also have a prior on θθθ, more on that later). The prior is very easy to estimate: just count the number of times y=0y = 0y=0 and y=1y = 1y=1 in the data and fit a binormial distribution.

  • In general, learning p(x,y∣θ)p(\mathbf{x}, y | \theta)p(x,y∣θ) might be very difficult - because x\mathbf{x}x consists of KKK features, and even if each feature is only binary, we have 2K2^K2K possible values of x\mathbf{x}x: we’ll never get a dataset big enough to see each value of x\mathbf{x}x even once as KKK gets large!

  • So, we make an assumption that the features {x1,…,xK}\left\{x_{1}, \dots, x_{K}\right\}{x1​,…,xK​} are conditionally independent given yyy. Mathematically, this means that:
    p(x1,x2∣y)=p(x1∣y)p(x2∣y)p\left(x_{1}, x_{2} | y\right)=p\left(x_{1} | y\right) p\left(x_{2} | y\right)p(x1​,x2​∣y)=p(x1​∣y)p(x2​∣y)

  • and not p(x)=p(x1,x2)≠p(x1)p(x2)p(\mathbf{x})=p\left(x_{1}, x_{2}\right) \neq p\left(x_{1}\right) p\left(x_{2}\right)p(x)=p(x1​,x2​)​=p(x1​)p(x2​)

  • Now, p(x∣y,θ)=∏k=1Kp(xk∣y)p(\mathbf{x} | y, \theta) = \prod_{k=1}^{K} p\left(x_{k} | y\right)p(x∣y,θ)=∏k=1K​p(xk​∣y) (the number of features increases only linearly, rather than exponentially as earlier)

  • With the assumption of binary classification and binary features: KKK features, 2 classes => 2K2K2K class conditional Bernoulli distributions. 111 Bernoulli distribution for the prior. Total 2K+12K+12K+1 parameters to be estimated, all which correspond to Bernoulli success prob θ∈[0,1]\theta \in [0,1]θ∈[0,1].

  • The MLE objective for Naive Bayesis:
    L(θ)=∑i=1Nlog⁡p(xi,yi∣θ)\mathcal{L}(\theta)=\sum_{i=1}^{N} \log p\left(\mathbf{x}^{i}, y^{i} | \theta\right)L(θ)=i=1∑N​logp(xi,yi∣θ)
    (Expand above with the conditional independence assumption)

  • In order to optimize the objective, we simply need to estimate each of the distributions p(xk∣y)p\left(x_{k} | y\right)p(xk​∣y) and the prior p(y)p(y)p(y). Each of these can be treated as a separate MLE problem.

  • To estimate the θ\thetaθ parameter for the prior p(y)p(y)p(y):
    p(y=j)=Count⁡(y=j)∑j′Count⁡(y=j′)p(y=j)=\frac{\operatorname{Count}(y=j)}{\sum_{j^{\prime}} \operatorname{Count}\left(y=j^{\prime}\right)}p(y=j)=∑j′​Count(y=j′)Count(y=j)​
    The denominator corresponds to the size of the dataset.

  • For each class-conditional feature xk∣yx_k | yxk​∣y, we can estimate it’s θ\thetaθ parameter as:
    p(xk=ℓ∣y=j)=Count⁡(xk=ℓ and y=j)∑ℓ′Count⁡(xk=ℓ′ and y=j)p\left(x_{k}=\ell | y=j\right)=\frac{\operatorname{Count}\left(x_{k}=\ell \text { and } y=j\right)}{\sum_{\ell^{\prime}} \operatorname{Count}\left(x_{k}=\ell^{\prime} \text { and } y=j\right)}p(xk​=ℓ∣y=j)=∑ℓ′​Count(xk​=ℓ′ and y=j)Count(xk​=ℓ and y=j)​
    The denominator corresponds to the number of xkx_kxk​ records which are of class jjj.

  • Training/Implementation of NB simply involves pre-computing these probabilities so that at test time, we just need to plug-in the values. In general, we work in the log space so that mulitplication of small probabilities doesnt result in computational underflow.

  • Prediction (Inference): given a new record X⋆\mathbf{X}^{\star}X⋆ and our model of p(x,y)p(\mathbf{x}, y)p(x,y), we have to determine the y⋆y^{\star}y⋆ that makes the observed X⋆\mathbf{X}^{\star}X⋆ most probable.
    y⋆=arg⁡max⁡yp(x⋆,y)y^{\star}=\arg \max _{y} p\left(\mathbf{x}^{\star}, y\right)y⋆=argymax​p(x⋆,y)

  • We can compute the likelihood value p(x⋆,y=j)=p(x⋆∣y=j)p(y=j)p\left(\mathbf{x}^{\star}, y=j\right) = p\left(\mathbf{x}^{\star}| y=j\right) p(y=j)p(x⋆,y=j)=p(x⋆∣y=j)p(y=j) for each possible class yyy.   • Failure Case: What if one of the class conditional probabilities p(xk=ℓ∣y=j)p\left(x_{k}=\ell | y=j\right)p(xk​=ℓ∣y=j) is zero for the test record.
